# AI Code Review Reports

This directory contains all code review reports generated by AI agents.

## Purpose

- **Audit Trail**: Track all code reviews performed by AI agents
- **Team Visibility**: Share findings and recommendations with the team
- **Knowledge Base**: Build historical reference of patterns and issues
- **Transparency**: Document AI decisions and reasoning

## Report Types

### 1. Code Review Reports
**Naming**: `code_review_YYYY_MM_DD_[description].md`

Generated when reviewing code changes, components, or specific files.

**Examples:**
- `code_review_2026_02_10_hooks_refactor.md`
- `code_review_2026_02_10_user_form_component.md`

### 2. PR Review Reports
**Naming**: `pr_review_[pr_name_or_number].md` or `pr_review_YYYY_MM_DD_[description].md`

Generated when reviewing Pull Requests before merge.

**Examples:**
- `pr_review_myp_11.md`
- `pr_review_2026_02_10_myp_11_and_styles.md`

### 3. Post-Commit Reviews
**Naming**: `post_commit_review_YYYY_MM_DD.md`

Generated when reviewing code after it's been committed.

### 4. Branch Readiness Reports
**Naming**: `YYYY_MM_DD_[branch_name]_readiness.md`

Generated when assessing if a branch is ready for merge.

**Examples:**
- `2026_02_10_myp8_branch_readiness.md`

## Naming Conventions

- **Format**: snake_case with lowercase
- **Date Format**: `YYYY_MM_DD` (ISO 8601)
- **Descriptive**: Include PR number, branch name, or feature description
- **Extension**: Always `.md` (Markdown)

### ✅ Good Examples
- `code_review_2026_02_10_myp_11_hooks.md`
- `pr_review_myp_45_css_modules.md`
- `post_commit_review_2026_02_10.md`

### ❌ Bad Examples
- `CodeReview-2026-02-10.md` (wrong case)
- `review.md` (not descriptive)
- `PR Review 45.md` (spaces, wrong case)

## Report Structure

All reports should follow the template defined in `.opencode/skills/code-review/SKILL.md`.

### Minimum Sections
1. **Summary** - Brief overview
2. **Files Reviewed** - List of files
3. **Issues Found** - Critical, Warnings, Suggestions
4. **Compliance Checklist** - Standards verification
5. **Action Items** - Next steps

### Optional Sections
- Fixes Applied
- Statistics
- References
- Additional Notes

## Automation

Reports are **automatically generated** by:
- `@code-review` agent (defined in `.opencode/agents/code-review.md`)
- `code-review` skill (defined in `.opencode/skills/code-review/SKILL.md`)

**MANDATORY**: Every code review session MUST produce a report in this directory.

## Usage

### Reading Reports
Reports are written in Markdown and can be viewed in:
- GitHub/Bitbucket web interface
- VS Code with Markdown preview
- Any Markdown viewer

### Finding Reports
```bash
# List all reports
ls -lt ai_docs/reports/*.md

# Find PR reviews
ls ai_docs/reports/pr_review_*.md

# Find code reviews from specific date
ls ai_docs/reports/*2026_02_10*.md

# Search content
grep -r "Critical" ai_docs/reports/
```

### Referencing in PRs
Link to reports in PR descriptions:
```markdown
## Code Review
AI review report: [ai_docs/reports/pr_review_myp_11.md](../ai_docs/reports/pr_review_myp_11.md)
```

## Retention Policy

- **Keep all reports** - They serve as audit trail and historical reference
- **No automatic deletion** - Reports are small text files
- **Archive old reports** (optional) - Can move to `ai_docs/reports/archive/YYYY/` if needed

## Contributing

When manually creating reports:
1. Follow the naming convention
2. Use the template from `.opencode/skills/code-review/SKILL.md`
3. Include all mandatory sections
4. Be specific and actionable
